## CUDA / 驱动

```bash
# 看驱动
nvidia-smi

# 看 PyTorch 实际用的 CUDA
python -c "import torch; print(torch.version.cuda, torch.cuda.is_available())"
```

PyTorch 2.5 默认装 CUDA 12.4 的 wheel。如果你驱动太老(显示 CUDA < 12),按 [pytorch.org](https://pytorch.org/get-started/locally/) 选对应 CUDA 11.8 的 wheel:

```bash
pip install torch --index-url https://download.pytorch.org/whl/cu118
```

## Unsloth 安装(Linux / WSL2 限定)

Unsloth 官方只支持 Linux + CUDA。Windows 原生跑不起来,**必须用 WSL2**。

```bash
# 在 WSL2(Ubuntu)里:
pip install "unsloth[colab-new]@git+https://github.com/unslothai/unsloth.git"
# 或最新 release:
pip install unsloth
```

如果你在 Windows 原生,跳过涉及 Unsloth 的 notebook(02_full_param_sft_liger.ipynb / 03_unsloth_lora.ipynb 等),改用 TRL 原生路径 —— 05 章 README 会标清楚每个 notebook 是否需要 Unsloth。

## bitsandbytes 装不上(Windows 高频问题)

bitsandbytes 0.44+ 已经有官方 Windows wheel。如果 `pip install bitsandbytes` 报错:

```bash
# 升级 pip 再装
python -m pip install --upgrade pip
pip install bitsandbytes --force-reinstall --no-cache-dir
```

实在不行用预编译 wheel:

```bash
pip install bitsandbytes-windows  # 注意是 fork 包,只在卡 0.44 前用
```

## vLLM Windows 跑不了怎么办

vLLM 官方不支持 Windows。三个绕开方案:
1. **WSL2 + CUDA**(推荐)—— 设备一致体验最好
2. **云 GPU**(autodl / Vast.ai / Lambda)只在第 04 章用 1-2 小时
3. **跳过 vLLM**,用 `transformers.generate()` + `torch.compile` 做 baseline 对比,八股该懂的都懂

## HuggingFace 国内访问慢

```powershell
$env:HF_ENDPOINT = "https://hf-mirror.com"
```

或永久设置:

```powershell
[System.Environment]::SetEnvironmentVariable('HF_ENDPOINT', 'https://hf-mirror.com', 'User')
```

镜像与官方 1:1 同步,`from_pretrained` / `snapshot_download` 自动走它。

## 内存炸了怎么办(SFT/DPO 章常见)

按效果递减依次启用:
1. `gradient_checkpointing=True` —— 拿计算换显存,最有效
2. 减小 `per_device_train_batch_size`(=1)+ 提高 `gradient_accumulation_steps`
3. 启用 AMP(bf16 优先,3000 系以上都支持)
4. 短 `max_seq_length`(SFT 1024 起够用)
5. 上 QLoRA(看第 02 章 04 节)
